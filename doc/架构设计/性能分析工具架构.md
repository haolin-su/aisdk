# 性能分析工具架构



## 架构设计


性能分析有以下层级：
1）最底层是各个芯片厂商SDK得出的各种资源获取接口，比如CPU/GPU/NPU/DSP，内存和显存，带宽等数据；
2）fibo sdk需要在此基础上进行各个厂商性能接口封装的抽象再封装成统一的接口，这样对外和代码内部都是统一的性能采集接口
也方便不同的同事写不同性能分析代码接口时进行抽象封装；
3）fibo sdk性能分析工具，可对外使用，也可直接http封装后给测试使用
4）在fibo sdk性能分析工具基础上，封装python接口给ai tools工具，作为其中的task之一
5）ai tools工具，可对外使用，也可直接http封装后给测试使用
6）ai tools工具，可对结果进行二次分析；
7）测试人员使用ai tools工具进行性能分析，并将结果上报给fibo sdk性能分析工具，fibo sdk性能分析工具再将结果上报给fibo sdk的性能分析平台
